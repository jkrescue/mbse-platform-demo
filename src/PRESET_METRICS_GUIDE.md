# 预置指标体系使用指南

## 概述

本系统为汽车行业MBSE工作流管理应用提供了完整的预置指标体系，包含17项核心指标和6个竞品对标数据，支持雷达图可视化对比和任务-指标关联追溯。

## 功能特性

### 1. 预置指标体系

包含5大类别共17项指标：

#### 性能指标 (5项)
- **续航里程** (PERF_RANGE): 目标600km，来源于客户需求和竞品对标
- **0-100km/h加速时间** (PERF_ACCEL): 目标5.8s，来源于市场竞争
- **最高车速** (PERF_TOP_SPEED): 目标180km/h，来源于法规要求
- **百公里能耗** (PERF_ENERGY_CONSUMPTION): 目标13.5kWh/100km
- **制动距离(100-0km/h)** (PERF_BRAKING): 目标36m，来源于安全法规

#### 成本指标 (3项)
- **整车成本** (COST_VEHICLE): 目标18万元
- **电池成本** (COST_BATTERY): 目标750元/kWh
- **三电系统成本** (COST_POWERTRAIN): 目标6.5万元

#### 可靠性指标 (3项)
- **电池循环寿命** (RELI_BATTERY_LIFE): 目标2000次
- **整车故障率** (RELI_FAILURE_RATE): 目标300 PPM
- **电机MTBF** (RELI_MOTOR_MTBF): 目标50000小时

#### 安全指标 (3项)
- **C-NCAP评分** (SAFE_CNCAP): 目标5星
- **电池碰撞安全** (SAFE_BATTERY_CRASH): 目标1级
- **热失控预警时间** (SAFE_THERMAL_WARNING): 目标5分钟

#### 合规性指标 (2项)
- **CLTC工况续航** (COMP_CLTC_RANGE): 目标620km
- **电磁兼容性** (COMP_EMC): 目标5级

### 3. 雷达图可视化

支持按类别对比项目目标与竞品指标：

- **性能指标雷达图**: 对比续航、加速、最高车速、能耗、制动等
- **成本指标雷达图**: 对比整车成本、电池成本、三电系统成本
- **可靠性指标雷达图**: 对比电池寿命、故障率、MTBF等
- **安全指标雷达图**: 对比C-NCAP评分、碰撞安全、热失控预警
- **合规性指标雷达图**: 对比CLTC续航、电磁兼容性

所有指标已归一化到0-100分制，便于不同量纲的指标对比。

## 使用流程

### 步骤1: 初始化指标体系

1. 进入项目详情页
2. 切换到"任务管理"标签页
3. 点击"加载预置指标体系"按钮
4. 在弹窗中查看指标概览和竞品数据
5. 点击"确认初始化"

初始化完成后，系统会创建17个指标和6个竞品对标数据。

### 步骤2: 创建任务并关联指标

1. 在任务管理页面点击"创建任务"
2. 填写任务基本信息（名称、描述、类型等）
3. 在"关联指标"部分，有两种方式选择指标：

   **方式A: 列表选择**
   - 切换到"指标列表"标签页
   - 按类别浏览指标（性能、成本、可靠性、安全、合规）
   - 勾选需要关联的指标
   - 支持按类别"全选"和"清除"

   **方式B: 雷达图对比**
   - 切换到"雷达图对比"标签页
   - 选择对比维度（性能、成本、可靠性等）
   - 查看项目目标与竞品的雷达图对比
   - 识别优势指标和差距指标
   - 返回"指标列表"选择相关指标

4. 完成其他任务信息填写
5. 点击"创建任务"

### 步骤3: 任务指标评估

任务创建后，需要对每个关联的指标进行详细评估：

1. 打开任务详情
2. 在"指标评估"面板中，对每个指标填写：
   - **来源评估**: 客户需求/法规要求/竞品对标/内部目标/技术约束/成本优化
   - **完成信心**: 很高/高/中/低/很低
   - **信心依据**: 说明为什么有这个信心水平
   - **最优性评估**: 最优/接近最优/可接受/需要改进/不可行
   - **最优性说明**: 为什么是这个最优性水平
   - **基线值/目标值/挑战值**: 设定三档目标
   - **风险因素**: 识别可能的风险
   - **技术挑战**: 识别技术难点

3. 保存评估信息

### 步骤4: 模型提交与验证

1. 在任务执行过程中，提交相关模型
2. 建立模型参数与指标的映射关系
3. 通过工作流执行验证指标达成情况
4. 查看工作流评估报告

### 步骤5: 指标追溯

完整的MBSE追溯链条：

```
项目指标 → 任务指标 → 模型参数 → 工作流执行 → 验证结果
```

可以追溯：
- 指标来自哪个任务
- 任务使用了哪些模型
- 模型通过哪些工作流验证
- 验证结果是否满足指标要求

## 指标归一化说明

雷达图中的指标归一化规则：

### 正向指标（值越大越好）
- 续航里程: 0-800km → 0-100分
- 最高车速: 150-220km/h → 0-100分
- 电池循环寿命: 1500-3000次 → 0-100分
- C-NCAP评分: 0-5星 → 0-100分

### 反向指标（值越小越好）
- 加速时间: 3-8s → 100-0分（归一化后反转）
- 能耗: 10-20kWh/100km → 100-0分
- 制动距离: 30-40m → 100-0分
- 成本类指标: 归一化后反转

## 自定义扩展

### 添加新指标

1. 在"指标管理"模块点击"创建指标"
2. 填写指标信息（名称、代码、类别、单位、目标值等）
3. 新指标会自动出现在任务关联指标列表中

### 添加竞品数据

1. 在"竞品对标"模块点击"添加竞品"
2. 填写竞品信息和各项指标值
3. 竞品数据会自动出现在雷达图对比中

### 调整指标目标

1. 在"指标管理"中找到要调整的指标
2. 点击"编辑"修改目标值、上下限等
3. 修改会影响所有使用该指标的任务

## 最佳实践

1. **项目启动时立即初始化指标体系**，建立指标基准
2. **任务创建时参考雷达图**，识别关键差距指标优先关联
3. **定期更新竞品数据**，保持对标数据的时效性
4. **充分评估指标可行性**，避免设定不切实际的目标
5. **建立完整的追溯链**，从指标到模型到工作流到验证
6. **利用权衡分析**，在相互冲突的指标间找平衡

## 技术架构

### 数据文件
- `/data/presetMetrics.ts`: 预置指标和竞品数据定义
- 包含指标归一化配置和雷达图类别配置

### 核心组件
- `MetricSelector`: 指标选择器（列表+雷达图）
- `MetricRadarChart`: 雷达图可视化
- `MetricRadarComparison`: 竞品对比雷达图
- `PresetMetricsInitializer`: 预置指标初始化器

### 服务层
- `MetricsService.initializePresetMetrics()`: 初始化预置指标
- `MetricsService.getCompetitorRadarData()`: 获取雷达图数据

## 常见问题

### Q: 是否可以修改预置指标？
A: 可以。初始化后的指标可以在"指标管理"中编辑，但不会影响预置数据源。

### Q: 重复初始化会怎样？
A: 会创建新的指标，不会覆盖已有指标。建议只初始化一次。

### Q: 如何删除不需要的指标？
A: 在"指标管理"中可以删除指标，但建议保留以保持追溯完整性。

### Q: 雷达图为什么某些竞品缺少数据？
A: 预置数据中该竞品可能没有该指标的数据，可以手动补充。

### Q: 如何自定义雷达图类别？
A: 修改 `/data/presetMetrics.ts` 中的 `RADAR_CHART_CATEGORIES` 配置。
