# 任务管理与指标下发 - 快速开始指南

## 功能入口

### 1. 访问任务管理

```
路径: 项目中心 → 选择项目 → 任务管理标签
```

1. 在左侧导航栏点击"项目管理"
2. 选择一个项目（如"A00级纯电动车开发"）
3. 在项目仪表板顶部点击"任务管理"标签

## 快速操作指南

### 创建任务并关联指标

#### 步骤1: 创建任务

1. 点击右上角"创建任务"按钮
2. 填写基本信息:
   - 任务名称: 例如"电池包能量密度优化"
   - 任务描述: 描述任务目标和范围
   - 所属阶段: 选择项目里程碑
   - 任务类型: 仿真/设计/测试等
   - 优先级: 紧急/高/中/低

#### 步骤2: 设置时间和人员

- 开始日期: 2025-11-01
- 截止日期: 2025-12-31
- 预计工时: 120小时
- 负责人: 从团队成员中选择
- 协作者: 可选多个协作人员

#### 步骤3: 关联指标 ⭐核心功能

- 从项目指标库中勾选相关指标
- 例如: "电池包能量密度 (ENERGY_DENSITY_001)"
- 可以选择多个指标
- 系统会提示: 任务创建后需要对每个指标进行详细评估

#### 步骤4: 创建成功

- 任务出现在看板上
- 可以看到任务卡片显示关联的指标数量

### 进行指标评估 ⭐核心功能

#### 什么时候评估？
任务开始阶段，在执行具体工作之前

#### 如何评估？

1. 点击任务卡片打开详情
2. 切换到"指标评估"标签
3. 点击"编辑评估"按钮
4. 填写以下内容:

**指标来源** (必填)
- 选项: 客户需求/法规要求/竞品对标/内部目标/技术约束/成本优化
- 来源详情: 具体说明，例如"客户要求续航里程500km，反推得出能量密度需求"

**完成信心** (必填)
- 选项: 非常高/高/中/低/非常低
- 信心依据: 说明为什么有这样的信心，例如"团队有类似项目经验，技术路径清晰"

**最优性评估** (必填)
- 选项: 最优/接近最优/可接受/需要改进/不可行
- 最优性说明: 例如"当前技术条件下可达到的较优水平，略低于行业领先水平"

**目标值设置**
- 基线值: 165 (当前水平)
- 目标值: 180 (必须达到)
- 挑战值: 190 (努力方向)

**风险与挑战** (可选但推荐填写)
- 风险因素: 例如"电芯供应商产能不足"
- 技术挑战: 例如"Pack效率提升空间有限"

5. 保存评估

### 提交模型并建立匹配关系 ⭐核心功能

#### 步骤1: 准备模型

在任务执行过程中开发或优化模型

#### 步骤2: 提交模型

1. 在任务详情中切换到"模型提交"标签
2. 点击"提交模型"按钮
3. 填写模型信息:
   - 模型名称: "电池包系统模型"
   - 模型版本: "v1.2"
   - 提交类型: 初始提交/更新/修订/最终版本
   - 模型描述: 描述模型功能
   - 变更日志: 记录本次提交的变更

#### 步骤3: 建立模型-指标匹配 ⭐重要

1. 点击"添加匹配"按钮
2. 为每个匹配关系填写:

**关联指标**: 选择任务中的指标

**匹配类型**:
- 直接输出: 模型直接输出该指标值
- 派生计算: 需要通过公式计算得到
- 约束条件: 模型的约束参数
- 验证手段: 用于验证指标的模型

**参数路径**: 
- 例如: `battery.energyDensity`
- 例如: `pack.thermalManagement.efficiency`

**计算公式** (如果是派生计算):
- 例如: `totalCapacity / totalWeight`

**匹配可信度**:
- 评估这个匹配关系的可靠程度
- 非常高/高/中/低/非常低

3. 可以添加多个指标匹配
4. 提交模型

#### 步骤4: 模型验证

- 架构师或项目经理查看模型提交
- 点击"通过"或"拒绝"按钮
- 填写验证意见

### 执行工作流并评估指标

#### 步骤1: 执行工作流

1. 在工作流管理中打开相关工作流
2. 配置并执行仿真工作流
3. 等待执行完成

#### 步骤2: 保存评估结果 (计划中)

当前需要手动在任务详情的"工作流评估"标签查看结果

未来将支持:
1. 工作流执行完成后显示"保存评估"按钮
2. 自动获取仿真结果
3. 对比目标值并计算偏差
4. 提供改进建议
5. 自动更新任务指标进度

#### 步骤3: 查看评估历史

1. 在任务详情切换到"工作流评估"标签
2. 查看所有评估记录
3. 对比不同时间点的结果
4. 查看改进趋势

## 看板视图说明

### 阶段列

- 每列代表一个项目阶段（里程碑）
- 显示阶段名称、目标日期、进度

### 任务卡片

**卡片头部**:
- 优先级图标（红/橙/黄/灰）
- 任务编号（如 TASK-001）

**任务名称**: 点击可查看详情

**状态标签**:
- 未开始（灰色）
- 进行中（蓝色）
- 审核中（紫色）
- 已完成（绿色）
- 阻塞（红色）

**关键信息**:
- 👤 负责人
- 📅 截止日期
- 🎯 关联指标数量
- 📦 提交模型数量

**进度条**:
- 任务整体进度
- 指标平均达成率

## 统计卡片说明

### 总任务数
当前项目的任务总数

### 进行中
状态为"进行中"的任务数量

### 已完成
状态为"已完成"的任务数量

### 完成率
已完成任务 / 总任务数

### 关联指标
所有任务关联的指标总数

### 提交模型
所有任务提交的模型总数

## 最佳实践

### 1. 指标评估的最佳时机
✅ 任务创建后立即进行评估
✅ 在执行具体工作前完成评估
❌ 不要在任务快结束时才评估

### 2. 信心评估的诚实原则
✅ 诚实评估，不要过度乐观
✅ 基于团队能力和资源实际情况
✅ 记录信心依据，方便后续复盘

### 3. 模型-指标匹配的准确性
✅ 确保参数路径准确无误
✅ 明确匹配类型（直接/派生/约束/验证）
✅ 如果是派生计算，写清楚公式

### 4. 工作流评估的及时性
✅ 工作流执行完立即保存评估
✅ 及时记录改进建议
✅ 定义明确的下一步行动

### 5. 进度更新的频率
✅ 建议每周更新一次指标进度
✅ 重大变化时及时更新
✅ 保持趋势信息的准确性

## 常见问题

### Q1: 如何选择合适的指标来源？
A: 根据实际情况选择:
- 客户明确提出的要求 → 客户需求
- 国家法规强制要求 → 法规要求
- 对标竞品得出的目标 → 竞品对标
- 团队内部设定的目标 → 内部目标
- 技术限制决定的约束 → 技术约束
- 成本优化的需要 → 成本优化

### Q2: 完成信心如何评估？
A: 考虑以下因素:
- 团队经验: 做过类似项目 → 信心高
- 技术成熟度: 技术路径明确 → 信心高
- 资源保障: 资源充足 → 信心高
- 时间充裕度: 时间充足 → 信心高
- 外部依赖: 依赖少 → 信心高

### Q3: 什么是最优性评估？
A: 评估指标设置是否最优:
- 最优: 行业领先水平，很难再提升
- 接近最优: 接近行业领先，有小幅提升空间
- 可接受: 满足基本要求，有较大提升空间
- 需要改进: 勉强能接受，必须改进
- 不可行: 无法实现，需要重新设定

### Q4: 模型参数路径怎么写？
A: 使用点号分隔的层级结构:
- 单层: `energyDensity`
- 多层: `battery.pack.energyDensity`
- 数组: `cells[0].voltage`
- 建议与实际模型结构保持一致

### Q5: 工作流评估的偏差怎么理解？
A: 
- 正偏差: 仿真值 > 目标值（好事）
- 负偏差: 仿真值 < 目标值（需改进）
- 偏差百分比: (仿真值 - 目标值) / 目标值 × 100%
- ±5%以内: 认为达标
- -5%~-10%: 接近目标
- <-10%: 低于目标
